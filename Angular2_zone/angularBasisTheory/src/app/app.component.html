¿Como puedo crear un componente?

De manera manual

  1. Crear una carpeta dentro de la carpeta "app" con el nombre de tu component [Es una buena praxis utilizar el nombre del componente].
  2. Dentro de la carpeta de nuestro nuevo component creamos un archivo "nombreNuestroComponente.component.ts": este archivo es la configuración en código typeScript de nuestro componente dentro de este archivo hay que exportar una clase que tenga como nombre "nombreNuestroComponenteComponent", para informar a angular de que esta clase es un componente se usa @Component , la cual se debe importar del paquete @angular/core, esta etiqueta "@component" contiene un objeto java con las siguientes propiedades:
    1. "selector: " -> Un selector, que permite inscrutarlo en html clasico como si fuera un tag. Hay que reseñar que hay 3 tipos de selectores uno es el selector clasico entre corchetes angulares como si fuera un elemento html para eso se pone solo el nombre del componente 
      selector:'app-server'
      también esta el selector por atributo 
      selector: '[app-server]'
      , en este caso el componente se asignara a un elemento html si aparece app-server como atributo en un div u otro elemento, por ejemplo 
       <div app-server>  </div>
      , finalmente tenemos el selector por clase 
      selector: '.app-server'
      todo elemento que tenga la clase "app-server" sera tratada como un componente
    2. "templateUrl: " -> Una url que sirve para indicar donde esta la marca html que es como la plantilla
    Ejemplo
        @Component({ 
            selector: 'app-server',
            templateUrl: './server.component.html'
        })
        export class ServerComponent {

        }
  3. Insertar en el archivo app.component.ts en la propiedad "declarations" el bombre del nuevo component e importar el modulo en este mismo archivo @import "nuestroComponente" from "localizacion clase componente"

Vinculación de datos entre markup HTML y archivo typescript asociado

  • String iterpolation (Salida de datos desde el typescript hacia el HTML template ): Entre llaves se puede poner directamente una variable del typescript asociado al componente o se puede poner expresiones que terminen devolviendo un String o incluso metodos, con la condición de que devuelvan siempre un String 
         { { variable de typeScript } }
  • Vinculación por propiedad o property biding(Salida de datos desde el typescript hacia el HTML template ): En una elemento html se pone entre corchetes rectangulares un determinado atributo, esto indica que el valor de ese atributo esta directamente vinculado a una determinada variable del typescript asociado al componente. De tal forma que si cambia el valor de esa variable en el typeScript en cualquier momento, también se cambiara en el HTML
  • Event binding: Vinculación de evento
